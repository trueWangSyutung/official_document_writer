# 党政公文写作助手

## 任务目标

### 核心功能
本Skill用于自动创作符合政治要求的公文内容，且符合GB/T 9704-2012规范。

### 什么是党政公文
党政机关公文是党政机关实施领导、履行职能、处理公务的具有特定效力和规范体式的文书，是传达贯彻党和国家的方针政策，公布法规和规章，指导、布置和商洽工作，请示和答复问题，报告、通报和交流情况等的重要工具。

### 技能目标
- 自动创作符合政治要求的公文内容，且符合GB/T 9704-2012规范
- 支持用户自定义关键词，生成相关公文内容
- 生成符合GB/T 9704-2012规范的Word文档
- 支持用户自定义公文标题、发文单位、日期、字号等元数据
- 提供数据分析支持，生成可视化图表

### 能力包含
- 自动生成符合政治要求的公文内容，且符合GB/T 9704-2012规范
- 支持用户自定义关键词，生成相关公文内容
- 生成符合GB/T 9704-2012规范的Word文档
- 支持用户自定义公文标题、发文单位、日期、字号等元数据
- 提供数据分析功能（同比增长、环比增长分析）
- 提供数据可视化功能（趋势图、对比图）

### 触发条件
- 用户需要创作符合要求的公文内容
- 用户提到"公文写作"、"党政公文"、"公文格式"
- 用户需要将Word文档格式化为国家标准
- 用户需要分析数据并生成报告

## 操作步骤

### 标准流程

**重要说明**：本技能支持"先生成预览，满意再发布"的模式。热点采集和内容生成阶段完全不需要任何配置。

#### 阶段一：内容生成（无需配置）

**默认模式说明**：如果用户没有特殊要求，默认生成**国央企公文**。

##### 1. 选择发文单位（可选）
- 如果用户没有指定发文单位，默认为**国央企**
- 如果用户指定其他发文单位，则按单位要求生成内容
- 常见发文单位类型：
  - 国央企
  - 政府机关
  - 事业单位
  - 社会团体

##### 2. 确定内容方向（无需配置）
- 智能体询问用户关注的关键词（如：AI、量化、新能源、数字经济、碳中和等）
- 智能体询问用户的发文类型：
  - **上行文**：请示、报告、议案
  - **下行文**：决定、通知、通报、批复、命令（令）、意见
  - **平行文**：函
  - **其他**：决议、公报、公告、通告、纪要

**行文关系说明**：
- **上行文**：向上级单位行文，如请示、报告等
  - 适用于：需要上级作出明确解释和答复的事项
  - 适用于：请求在执行上做某些变通处理
  - 适用于：需要上级帮助解决本单位的某一具体问题
  - 适用于：事关重大、涉及面广，需上级审批、协调

- **下行文**：向下级单位行文，如通知、意见、通报、批复等
  - 适用于：批转下级机关的公文
  - 适用于：转发上级和不相隶属机关的公文
  - 适用于：传达要求下级机关办理、执行的工作事项
  - 适用于：传达需要下级和有关单位周知的事项
  - 适用于：任免人员

- **平行文**：同级单位或不相隶属的单位之间行文
  - 一般用函等文种
  - 适用于：商洽工作、询问和答复问题
  - 适用于：向有关主管部门请求批准

**数据分析支持**：
- 如果需要数据分析，智能体会调用 `scripts/analyze.py` 进行分析
- 分析结果包含：
  - 相关性分析
  - 同比增长（同比增长率）
  - 环比增长（环比增长率）
- 必要时可以调用 `scripts/draw_graph.py` 进行可视化分析
- 支持生成：
  - 趋势图
  - 对比图
  - 增长率柱状图

##### 3. 生成完整内容（无需配置）
智能体根据选择的热点和目标平台，生成包含以下元素的完整内容：

**3.1 标题生成**：
- 根据发文类型和关键词，生成符合GB/T 9704-2012规范的标题
- 完整的公文标题依次由以下三部分组成：
  1. **发文机关名称**：撰写公文的单位名称
  2. **发文事由**：准确概括公文的主要内容，一般由介词短语"关于……"表示
  3. **文种**：公文的类型名称

**标题格式示例**：
```
发文机关 + 关于 + 事由 + 的 + 文种
国央企  + 关于 + XXX事项 + 的 + 决定
```

**注意事项**：
- 发文机关名称有时可省略
- "关于"不可省略
- 事由不可省略，要概括得准确、简要、明了
- 文种不可省略
- 标题中一般不使用标点符号

**标题模板参考**：
- 详见 [references/title-templates.md](references/title-templates.md)

**3.2 写作指南**：

**上行文写作指南**：
- 根据发文类型和关键词，生成符合GB/T 9704-2012规范的上行文
- 适用于：
  - 需要上级作出明确解释和答复
  - 不经有关部门批准，本单位无权决定的事项（按规定请示）
  - 需要上级作出明确指示或提出解决的办法（请示新政策）
  - 请求在执行上做某些变通处理，有待上级定夺（变通性）
  - 需要上级帮助解决本单位的某一具体问题和实际困难
  - 事关重大、涉及面广，为防止工作出现失误，需上级审批、协调（请求审批）
  - 向上级机关汇报工作
  - 向上级机关反映情况
  - 答复上级机关的询问
  - 报送材料

**下行文写作指南**：
- 根据发文类型和关键词，生成符合GB/T 9704-2012规范的下行文
- 适用于：
  - 批转下级机关的公文
  - 转发上级和不相隶属机关的公文
  - 传达要求下级机关办理、执行的工作事项
  - 传达需要下级和有关单位周知的事项
  - 任免人员

**平行文写作指南**：
- 根据发文类型和关键词，生成符合GB/T 9704-2012规范下的平行文
- 具体格式参考 [references/write.pdf](references/write.pdf) 中的公文基础知识与行文规范

**3.3 Word排版**：
- 根据发文类型和关键词，生成符合GB/T 9704-2012规范下的Word文档
- 具体格式参考：
  - [references/write.pdf](references/write.pdf) - 公文基础知识与行文规范
  - [references/format.pdf](references/format.pdf) - Word排版规范
  - [references/党政机关公文格式 (1).pdf](references/党政机关公文格式 (1).pdf) - 党政机关公文格式国家标准
- 调用 `scripts/format.py` 进行Word排版
- 输出格式：Word文档（.docx）

**GB/T 9704-2012 排版规范要点**：
- **页面设置**：A4纸（210mm × 297mm）
- **页边距**：
  - 上边距：3.7cm
  - 下边距：3.5cm
  - 左边距：2.8cm
  - 右边距：2.6cm
- **字体**：
  - 标题：方正小标宋简体，二号字（22pt）
  - 正文：仿宋_GB2312，三号字（16pt）
- **行间距**：固定值28磅
- **首行缩进**：2字符（0.85cm）


**3.4 附件**：
- 如果正文包含附件，则根据发文类型和关键词，生成符合GB/T 9704-2012规范的附件
- 并且附到Word文档中
**附件格式**：
- **附件名称**：附件 + 序号 + 附件名


**注意事项**：
- 如遇到需要姓名的情况，请使用从 [references/name.json](references/name.json) 中随机抽取的姓名。

##### 4. 预览与调整（无需配置）
- 将生成的完整文章（Word格式）展示给用户
- 根据用户反馈，调整文章内容
- 循环以上步骤，直到用户满意为止

#### 阶段二：数据处理（可选）

如果用户需要数据分析支持，可以执行以下操作：

##### 数据分析
```bash
python scripts/analyze.py [数据文件] [输出文件]
```

**示例**：
```bash
python scripts/analyze.py data.csv output.csv
```

**支持格式**：
- CSV格式（.csv）
- Excel格式（.xlsx, .xls）

**输出指标**：
- 原始数据
- 上月数值（环比）
- 去年同期数值（同比）
- 同比增长（%）
- 环比增长（%）

##### 数据可视化
```bash
python scripts/draw_graph.py [数据文件] [输出文件] [图表类型]
```

**图表类型**：
- `trend`：趋势图（默认）
- `comparison`：对比图（包含增长率）

**示例**：
```bash
# 生成趋势图
python scripts/draw_graph.py data.csv trend.png trend

# 生成对比图
python scripts/draw_graph.py data.csv comparison.png comparison
```

**支持格式**：
- PNG（推荐）
- JPG
- PDF

#### 阶段三：文档格式化（可选）

如果用户已有Word文档需要格式化，可以直接使用：

```bash
python scripts/format.py [输入文件] [输出文件]
```

**示例**：
```bash
python scripts/format.py draft.docx formatted.docx
```

##### 添加红头标识

为文档添加符合GB/T 9704-2012规范的红头标识：

```bash
python scripts/format.py [输入文件] [输出文件] --red-head \
    --org "发文单位名称" \
    --title "文件标题" \
    --number "文件编号" \
    --date "发文日期"
```

**参数说明**：
- `--red-head`: 添加红头标识
- `--org`: 单位名称（默认：国央企）
- `--title`: 文件标题
- `--number`: 文件编号（如：国央企〔2024〕1号）
- `--date`: 发文日期

**示例**：
```bash
python scripts/format.py input.docx output.docx --red-head \
    --org "国央企" \
    --title "关于加强数据安全管理的通知" \
    --number "国央企〔2024〕5号" \
    --date "2024年1月15日"
```

**红头格式规范**：
- 单位名称：黑体，二号字，红色，居中
- 文件标题：方正小标宋简体，二号字，红色，居中
- 文件编号：仿宋_GB2312，三号字，红色，右对齐
- 日期：仿宋_GB2312，三号字，红色，右对齐
- 分隔线：红色横线，居中

## 注意事项

### 默认模式（无特殊要求时）
- 默认生成国央企公文
- 发文机关名称：国央企
- 发文事由：关于XXX
- 发文类型：决定

### 标题撰写规范

**基本要求**：
- **准确**：标题应当准确反映公文的主要内容
- **简要**：避免使用冗长、啰嗦的表述
- **规范**：使用规范的公文用语
- **完整**：除非特殊情况，一般应包含完整的标题要素

**常见错误**：
- ❌ 标题冗长：国央企关于认真贯彻落实党中央国务院关于加快推进新能源产业发展的若干意见的通知
- ✅ 规范标题：国央企关于加快推进新能源产业发展的通知

- ❌ 缺少要素：关于申请经费的
- ✅ 规范标题：国央企关于申请2024年度科研经费的请示

- ❌ 用语不当：关于搞一搞年终总结的通知
- ✅ 规范标题：关于做好2024年度工作总结的通知

### 行文规范

**上行文注意事项**：
- 请示应当一文一事
- 不得在报告等非请示公文中夹带请示事项
- 应当标注签发人姓名
- 应当使用正确的称谓

**下行文注意事项**：
- 通知应当明确执行要求和时限
- 通报应当事实清楚、定性准确
- 批复应当针对请示事项逐一答复

**平行文注意事项**：
- 函的用语应当礼貌、得体
- 应当明确询问或答复的事项

### 排版规范

**页面设置**：
- 纸张：A4（210mm × 297mm）
- 打印方向：纵向
- 页边距：上3.7cm，下3.5cm，左2.8cm，右2.6cm

**字体规范**：
- 标题：方正小标宋简体，二号字，居中
- 正文：仿宋_GB2312，三号字
- 数字：Times New Roman

**段落格式**：
- 行间距：固定值28磅
- 首行缩进：2字符
- 段前间距：0
- 段后间距：0

### 安全原则
- 确保用户在发布过程中所使用的所有图片、视频、音频等资源均符合版权要求
- 确保用户所使用的素材的版权归用户所有
- 生成的公文内容应当符合政治要求和法律法规
- 不得生成虚假或误导性的公文内容

## 故障排除

### 常见问题

**Q1: 运行脚本时提示 "ModuleNotFoundError"？**
```
A: 缺少必要的Python包。运行以下命令安装依赖：
pip install -r requirements.txt
```

**Q2: Word文档字体显示不正确？**
```
A: 确保系统已安装所需的字体文件：
- 方正小标宋简体
- 仿宋_GB2312
- 黑体
```

**Q3: 数据分析脚本无法读取文件？**
```
A: 请检查：
1. 文件路径是否正确
2. 文件格式是否为CSV或Excel
3. 文件是否包含date/日期列和数值列
```

**Q4: 图表中文字体显示为方块？**
```
A: 确保matplotlib配置了中文字体。可以修改scripts/draw_graph.py中的字体设置。
```

**Q5: format.py格式化失败？**
```
A: 请检查：
1. 输入文件是否为有效的Word文档（.docx）
2. 文件是否被其他程序占用
3. 是否有写入权限
```

**Q6: 如何处理大型文档？**
```
A: 对于大型文档，建议：
1. 分批处理内容
2. 增加Python的内存限制
3. 使用流式处理（如果支持）
```
